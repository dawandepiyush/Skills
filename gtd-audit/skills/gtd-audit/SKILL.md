# GTD Audit Skill — Complete Documentation

## Overview
The GTD Audit is a collaborative task review tool that evaluates lists against five core Getting Things Done principles: Capture, Clarify, Organize, Reflect, and Engage.

## Task Acquisition (Priority Order)
1. **User-specified source** — Uses named apps if matching MCP tools exist
2. **Connected MCP/todo tools** — Scans for Things 3, Todoist, Notion, Asana, Linear, TickTick, OmniFocus, or Reminders
3. **Chat history** — Extracts pasted lists or markdown checklists
4. **Uploaded files** — Reads .txt, .md, .csv, or export formats
5. **User input** — Requests manual entry if no other source available

## Two Operating Modes

**Mode A (Full Audit):** Runs all five principles when user requests general review, weekly audit, or "how am I doing"

**Mode B (Deep-Dive):** Focuses on specific principle when user names one (e.g., "clarify these" or "what should I work on")

## The Five Principles

**Capture:** Flags uncaptured sub-tasks and incomplete project breakdowns

**Clarify:** Identifies vague verbs, noun-only tasks, and unclear definitions of done

**Organize:** Surfaces unassigned tasks, misplaced items, and duplicates

**Reflect:** Highlights overdue, stale, overloaded, and unfollowed-up tasks

**Engage:** Assesses clarity of current priorities and time/energy alignment

## Output Format
Full audits deliver a GTD Health Report with scores and flagged items per principle, concluding with a recommended deep-dive focus.
